# FastBargaining: Fast Solution of Dynamic Intra-Household Bargaining Models

This repository implements a dynamic intra-household bargaining model in which couples and singles make endogenous choices over time allocation (market work, home production) and consumption. The core contribution is the use of the **interpolated Endogenous Grid Method (iEGM)**, which solves the model roughly 50 times faster than standard Value Function Iteration while maintaining comparable accuracy.

The model is built on the [EconModel](https://github.com/NumEconCopenhagen/EconModel) framework and uses [consav](https://github.com/NumEconCopenhagen/consav) for interpolation and numerical utilities. The inner optimization loop is compiled to C++ via [NLopt](https://nlopt.readthedocs.io/).

## Environment setup

Python 3.12 is required. On first run, EconModel will automatically compile the C++ solver
kernels — a C/C++ compiler must be available (MSVC on Windows, gcc/clang on Linux/macOS).

### Option A — uv

```bash
uv venv
uv sync
```

Activate the environment:

```bash
# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate
```

### Option B — pip + venv

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate
# Linux / macOS
source .venv/bin/activate

pip install -r requirements.txt
```

### Option C — Anaconda / conda

```bash
conda env create -f environment.yml
conda activate fastbargaining
```

## Replicating results

### Illustration
The `Illustration/` folder contains a simplified unitary household model and a Jupyter notebook that walks through its solution. This replicates the results from the simple model.

### Application
All notebooks in the `Application/` folder should be run in order. Open them in
Jupyter Lab or VS Code.

| Notebook | Purpose | Output |
|----------|---------|--------|
| `01. Calibration.ipynb` | Estimate model parameters by matching empirical moments via Nelder-Mead. | `par_files/calibrated_par.pkl` |
| `02. Comparisons.ipynb` | Compare solution algorithms (iEGM, EGM, VFI) on policy functions and value functions. | Figures in `output/` |
| `03. Timings.ipynb` | Benchmark iEGM accuracy and speed across consumption grid sizes. | Timing/accuracy table (`output/`) |
| `04. Counterfactuals.ipynb` | Simulate a counterfactual scenario. | IRF plots and inequality tables in `output/` |

Notebooks 2–4 can be run independently once `par_files/calibrated_par.pkl` exists. The
`output/` directory is created automatically if it does not exist.

## Appendix: Maintaining dependency files

### Updating `requirements.txt`

`requirements.txt` is generated by uv from `pyproject.toml`. After adding or upgrading
packages in `pyproject.toml`, regenerate it with:

```bash
uv lock
uv export --no-hashes -o requirements.txt
```
